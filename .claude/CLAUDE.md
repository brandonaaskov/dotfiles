## JavaScript/Typescript
- No semicolons
- Function arguments should be typed
- Arguments should only have one type
- If there's a big if block and a very small else block, just return early with the inverse check instead of indenting a bunch of code for no reason

## Golang
- When prototyping and in the early stages of a project, runtime errors are okay os opposed to thorough error handling
- When you decide that the issue is "simpler", make sure to revert your previous changes that overcomplicated things. Don't leave trash behind.
